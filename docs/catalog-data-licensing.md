# Catalog data licensing check (backlog_week1.md 0.2)

Day-one gate per `docs/build-roadmap.md`/`docs/backlog_week1.md`: confirm the license/ToS of the chosen open sources before writing any import code (`scripts/import-catalog`, task 4.1). Checked directly against each source's own policy pages and, for Commons, the live API — not assumed. Scope checked: US cents (Lincoln Wheat Cents 1909–1958, per the Day 1 PM fixture already pulled) plus the two other seed-template candidates already identified in `CLAUDE.md`'s v1 log (State Quarters, Morgan Dollars), since the same rules apply across all three.

## 1. Wikipedia text (mintage lists, year/mint-mark/variety data)

- **License:** CC BY-SA 4.0 (pre-2009 content also under GFDL for compatibility). [Wikipedia:Reusing Wikipedia content](https://en.wikipedia.org/wiki/Wikipedia:Reusing_Wikipedia_content).
- **Obligation:** attribution (a hyperlink/URL to the source page, or an author list) and share-alike on any *reused Wikipedia content itself*. The policy doesn't carve out tables/facts differently from prose.
- **What actually lands in `coins`:** not Wikipedia's prose or table markup — just the underlying facts (which year/mint-mark/variety combinations exist). Facts are not independently copyrightable under US law (*Feist Publications v. Rural Telephone*, 1991) regardless of which source first published them; only a sufficiently original *compilation or expression* of facts is protected, and a normalized `(country, denomination, year, mint_mark, variety)` row is exactly the kind of non-original compilation that doctrine doesn't reach. This was already the working assumption in the Day 1 PM fixture ([scripts/import-catalog/fixtures/README.md](../scripts/import-catalog/fixtures/README.md)) — confirmed here, not just asserted.
- **Belt-and-suspenders anyway:** task 5.2's one-line Wikipedia attribution note in the `apps/web` footer (CC BY-SA credit) stays in the plan regardless, since it's a one-sentence cost that closes the question outright rather than relying solely on the facts-aren't-copyrightable argument.
- **Automated access (API:Etiquette / Wikimedia Terms of Use §12):** a descriptive `User-Agent` with contact info is mandatory ("you may be IP-blocked without notice" otherwise) — format `AppName/version (contact URL or email)`. No hard rate limit on reads, but requests should be sequential, not parallel, and batched (`|`-separated titles or generators) where possible. No API key needed for public read access. **Action for task 4.1:** the import script must set a real `User-Agent` (e.g. `coin-collector-companion/0.1 (https://github.com/<repo>; ratnikov.am@gmail.com)`), matching the one used ad hoc during this check.

## 2. Wikimedia Commons images

- **Accepted licenses on Commons:** public domain, CC0, CC BY (any version), CC BY-SA (any version), a few other free licenses. Non-commercial/no-derivative licenses (CC BY-NC, CC BY-ND, fair-use claims) are **not allowed to be hosted on Commons at all** — so anything actually present on Commons already cleared that bar; the remaining question is only *which* free license applies, since CC BY/BY-SA carry an attribution obligation that CC0/public domain don't.
- **Per-image machine-readable check — confirmed live against real files**, via `action=query&prop=imageinfo&iiprop=extmetadata`. Relevant `extmetadata` fields (verified against three real Commons coin images: `File:1909-S VDB Lincoln cent obverse.jpg`, `File:1909 US Penny.jpg` (US Mint credit), `File:1893-S Morgan dollar obverse.jpg`, all `License: pd`/`AttributionRequired: false`; and `File:1909 V.B.D Lincoln Cent Reverse Graphic.jpg`, `License: cc0`):
  - `License` — short machine code (`pd`, `cc0`, `cc-by-sa-3.0`, etc.)
  - `LicenseShortName` — human-readable (`"Public domain"`, `"CC0"`, …)
  - `Copyrighted` — `"True"`/`"False"` string
  - `AttributionRequired` — `"true"`/`"false"` string
  - `LicenseUrl`, `Artist`, `Credit` — populated for attribution when required
- **Import script gate (task 4.1), now concrete:** only import an image where `extmetadata.Copyrighted.value === "False"` (covers `pd`/public-domain-dedication cases) **or** `extmetadata.AttributionRequired.value === "false"` (covers CC0 and any other no-attribution-needed free license) — matching `system-design_v2.md` §4.3's existing "only pulls images tagged public domain or a license the app can satisfy without per-image UI attribution" line, now with the actual field names instead of a placeholder description. Anything with `AttributionRequired: "true"` (CC BY/BY-SA) gets skipped for v1 per that same design-doc line, since there's no per-image attribution UI planned. Store `image_source`/`image_license` from `Credit`/`LicenseShortName` regardless, for the images that do get imported.
- **Gotcha this check surfaced, not assumed:** a US Mint-credited image (`File:1909 US Penny.jpg`, `Artist: United States Mint`) is public domain — but **not every US-Mint-sourced image on Commons necessarily is** (see §3 below). This is exactly why the check has to be per-image via `extmetadata`, never a blanket "it's from the Mint, so it's PD" shortcut, even for images that look Mint-sourced.
- **API etiquette:** same User-Agent/rate-limit rules as §1 (same Wikimedia API infrastructure). `extmetadata` is documented as an "expensive" property — request it for a handful of images at a time, not in one giant batch.

## 3. US Mint publications (usmint.gov)

- **usmint.gov's own copyright page is explicit that PD is not a blanket assumption:** "you should not assume anything on this Web site is necessarily in the public domain." Most Mint-produced coin/medal designs *are* public domain as federal-employee works, but designs based on third-party-licensed or -assigned IP are copyrighted and excluded — this is exactly the same distinction Wikimedia Commons' own vetting page for US coins draws (see below), which is reassuring: two independent sources agree on where the line is.
- **Concrete, checkable rule (via [Commons:Determining if U.S. coins are free to use](https://commons.wikimedia.org/wiki/Commons:Determining_if_U.S._coins_are_free_to_use)):**
  - Any US coin **published without a copyright notice before March 1, 1989** is public domain — US coins have never carried a copyright notice, so this cutoff applies cleanly. **Lincoln Wheat Cents (1909–1958) and Morgan Dollars (1878–1921) are both well before this cutoff — confirmed safe as public domain designs**, no case-by-case check needed for those two templates.
  - Designer status matters for anything closer to or past that line: Mint-staff titles ("Sculptor-Engraver," "Chief Engraver," "Medallic Artist") = employee work = PD; "Artistic Infusion Program" or outside competition winners = contractor/third-party = copyrighted. Relevant for a future State Quarters template (1999–2008, past the 1989 cutoff) — quarter *reverse* designs went through an approval process involving both Mint staff and state-nominated artists, so that template needs the actual per-design check the Commons page describes, not the blanket pre-1989 pass Lincoln Cents/Morgan Dollars get. **Flagging this explicitly rather than assuming** — worth a dedicated pass before importing State Quarters, separate from this Day 1 gate covering the initial US-cents scope.
  - **Photographs of coins carry their own, separate copyright** even when the coin design itself is public domain — "sufficient originality to be a separately copyrightable derivative work." This is why the per-image Commons `extmetadata` check in §2 is still required for every image regardless of the underlying coin's design-copyright status; a PD coin design doesn't make every photo of it automatically reusable.
- **Automated access to usmint.gov itself: not viable, and not needed.** Every fetch attempt against `usmint.gov` in this check (`/policies`, `/about/policies`, `/privacy-policy`, `/terms.html`, `/learn/history/copyright-information`) returned `403 Forbidden`, including through the redirect from `catalog.usmint.gov/terms.html` — consistent with `.gov` bot-protection blocking non-browser traffic outright. This matches what the v1 build already did in practice (`CLAUDE.md`'s changelog: the seed templates were "cross-checked against standard Red Book mint-mark-by-year tables," a manual process, not a scripted fetch) — **task 4.1's import script should not plan on scripted access to usmint.gov at all**; treat US Mint publications as a manual cross-reference source (as v1 already did), with Wikipedia as the only source the script actually fetches from programmatically.

## Net result

- **Cleared to proceed with import code (task 4.1) for the current US-cents scope (Lincoln Wheat Cents) and, by the same pre-1989 rule, Morgan Dollars**: mintage/identity facts from Wikipedia (facts aren't copyrightable; footer attribution note as a belt-and-suspenders extra, task 5.2), images gated per-image through Commons' `extmetadata` (`Copyrighted`/`AttributionRequired` fields, confirmed live), US Mint publications used only as a manual cross-reference, never fetched programmatically.
- **Not cleared yet, flagged for a separate pass before it's imported:** State Quarters (1999–2008) — past the pre-1989 blanket-PD cutoff, needs the per-design employee-vs.-contractor check the Commons guidance page describes, not assumed clear by this gate.
- **Action items carried into task 4.1:** set a real `User-Agent` with contact info on every Wikimedia API request; implement the `Copyrighted`/`AttributionRequired` gate exactly as described in §2 rather than a coarser heuristic; do not attempt scripted fetches against `usmint.gov`.
